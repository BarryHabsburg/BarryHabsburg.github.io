Basic SQL to Retrieve and Create Tables


     This article will cover the most commonly used PROC SQL statements, and will provide basic examples for each. The SQL Nomenclature: Table (e.g. datasets), Rows (e.g. observations) and Columns (e.g. variables); this nomenclature will be import to comprehend the sematic of PROC SQL. Lets break down the PROC SQL statement:

          
PROC SQL;
SELECT column list
FROM table list
WHERE condition list
GROUP BY column list
ORDER BY column list
;
quit;

The semantics of SELECT, it is used to "select" the desired columns from the table (e.g. dataset); it is followed by FROM which specifies the dataset to select from and it location in the database. As in any programming language conditional statements are fundamental, and consequently, WHERE is that conditional statement in PROC SQL. While GROUP BY is the grouping statement and ORDER BY the data arranging statement.
     Many of the examples in this article utilized either the somedata.dat dataset and cars.dat dataset from SAS Essentials - Mastering SAS for Data Analytics, 2nd edition by Alan C. Elliot and Wayne A. Woodward; so will make the proper citation here, and below in the citation segment. An additional note is that we used SAS studio through SAS Academics On-Demand to create and run these examples. For the example, we created a SAS library with LIBNAME and named it SASDATA. In this first demonstration * was used in SELECT, so to return the full dataset to the result page.

            
Proc SQL;
    SELECT * FROM MYSASLIB.SOMEDATA
    ;
QUIT;


Making use of the ORDER BY statement, we can order the data by the values found in ID, and furthermore, it could be made in descending order.

            
PROC SQL;
    SELECT * FROM SASDATA.SOMEDATA
    ORDER BY ID DESC
    ;
QUIT;


To demonstrate a conditional statement with WHERE, we implemented the following conditional: GENDER contains "Female", which only return observations with "Female" as value in GENDER.

            
PROC SQL;
    SELECT * FROM SASDATA.SOMEDATA
    WHERE GENDER contains "Female"
    ORDER BY ID
    ;
QUIT;


Limiting variables outputted can be accomplished by replacing the * with typed names of the column variables from the given table; the names should be seperated by commas. If one want to recode a variable, one can use CASE. In the case of somedata.dat, we used CASE to recode "A" as "TRT 1" and "B" as "TRT 2", otherwise the value was recoded as "CONTROL".

            
PROC SQL;
    SELECT ID, GP, AGE, GENDER,
    CASE GP
        WHEN "A" THEN "TRT 1"
        WHEN "B" THEN "TRT 2"
        ELSE "CONTROL"
    END as TREATMENT
    FROM SASDATA.SOMEDATA
    ORDER BY ID
    ;
QUIT;


Note that there is a comma after "GENDER", that is because we were creating a new variable denoted TREATMENT. Suppose we needed to recode a numeric variable; then we can "<", "<", or "=" operators to create the conditional statements.

            
PROC SQL;
    SELECT *,
    CASE 
        WHEN AGE <= 13 THEN "KID"
        ELSE "TEEN"
    END as STATUS
    FROM SASDATA.SOMEDATA
    ;
QUIT;


Yet again, there is a comma after SELECT *, since CASE is creating a new variable named STATUS. The resulting table from PROC SQL will contain all variables plus the new variable STATUS. Proceeding on, if one desire to create a new table (e.g. dataset), one would use CREATE TABLE statement followed with the destinated working directory, or SAS library; Otherwise the PROC SQL statement retains all it functionality.

            
PROC SQL;
    CREATE TABLE SASDATA.FEMALES AS
    SELECT ID,GP,AGE from SASDATA.SOMEDATA	
    WHERE GENDER CONTAINS "Female"
    ORDER by ID
    ;
QUIT;

PROC SQL;
    SELECT * FROM SASDATA.FEMALES
    ;
QUIT;


See SAS Log for results - when you create a table, it is not displayed in the viewer. CREATE goes before the SELECT - creates SAS data set, no table displayed. From SAS LOG: NOTE: Table WORK.FEMALES created, with 30 rows and 3 columns.

            
PROC SQL;
    CREATE VIEW SASDATA.FEMALES AS
    SELECT ID,GP,AGE from SASDATA.SOMEDATA	
    WHERE GENDER CONTAINS "Female"
    ORDER by ID
    ;
QUIT;

PROC PRINT LABEL data=SASDATA.females2;
RUN;

UnlikeCREATE TABLE which creates a file in the working directory/SAS library, CREATE VIEW does not contain actual data and therefore cannot be stored. Rather, CREATE VIEW> describes the data stored in other file structures – in this case a SQL Query. To view the view table, one can use PROC PRINT LABEL which returned a printed table in the view.
     To select the distinct observations from a table, then replace VIEW with DISTINCT.

            
PROC SQL;
    SELECT DISTINCT SUBJECT
    FROM SASDATA.COMPlICATIONS;
QUIT;


Let perform a simple a calculation on a small dataset named ROOMSIZE.

            
DATA ROOMSIZE;
    INPUT ROOM $ WIDTH LENGTH @@;
    DATALINES;
    LIVING   14 22 DINING   14 12 BREAKFAST 10 12
    KITCHEN  12 16 BEDROOM1 18 12 BEDROOM2  12 14
    BEDROOM3 13 16 BATH1     8 12 BATH2      7 10
    BATH3     6  8 GARAGE   23 24 
    ;
RUN;

Proc SQL;
    SELECT *,
    ROUND(WIDTH*LENGTH, 0.01) FORMAT=6.2 AS AREA
    FROM ROOMSIZE
    ;
QUIT;


More calculations with a GROUP BY statement.

            
Proc SQL;
    CREATE TABLE TIMEADJ AS 
    SELECT GP,COUNT(GP) AS N,TIME1,TIME2,MAX(TIME1) FORMAT=6.2 AS TIME1MAX,
    ROUND(TIME2/(CALCULATED TIME1MAX)*100,0.01)
    FORMAT=6.2 AS TIME2ADJ
    FROM SASDATA.SOMEDATA
    GROUP BY GP;
QUIT;

PROC SQL; SELECT * FROM TIMEADJ; QUIT;


Note there are THREE variables created by calculation step, which was variables: N, TIME1MAX and TIME2ADJ. Use this PROC SQL; SELECT *, FROM TIMEADJ; QUIT; to view the table that was just created.

Basic SQL Operators for Combining Datasets

Combine two or more queries in various ways by using the following set operators (there are more than these):


Operators Descriptions
Union produces all unique rows from both queries.
Except produces rows that are part of the first query only.
Intersect produces rows that are common to both query results.
Outer Union concatenates the query results.


To demonstrtate through example the SQL operators that can combine tables together, we generated two tables, OLD1 and OLD2. The first append operator that will be presented will be UNION. Union appends, keeps only unique rows (e.g, observations).

                
DATA OLD1;
    INPUT SUBJ $ AGE YRS_SMOKE;
    DATALINES;
    001 34 12
    003 44 14
    004 55 35
    006 21 3
    011 33 11
    ;
RUN;
    
DATA OLD2;
    INPUT SUBJ $ AGE YRS_SMOKE MARRIED;
    DATALINES;
    006 21 3 .
    011 33 11 1
    012 25 19 0
    023 65 45 1
    032 71 55 1
    ;
RUN;
    
PROC SQL;
    SELECT * FROM OLD1
    union
    SELECT * FROM OLD2;
QUIT;


This is the same code from before - only difference is the duplicated records in the data sets. One row 6 is unique. Two row 11's are unique. (UNION keeps all unique.)
     UNION ALL keeps all rows, regardless of the rows respective uniqueness.

                
PROC SQL;
    SELECT * FROM OLD1
    union all
    SELECT * FROM OLD2;
QUIT;


Except only keeps the data from the first data set that are not in the 2nd data set (but all column variables are used).

                
PROC SQL;
    SELECT * FROM OLD1
    except
    SELECT * FROM OLD2;
QUIT;


Suppose there was a duplicate record 006 in the first data set; using EXCEPT record 6 would not appear in the result because there is a record 6 in the second data set. If you want a duplicate record that is not a duplicate matched in the 2nd data set to appear in the result, use EXCEPT ALL.

                
PROC SQL;
    SELECT * FROM OLD1
    except all
    SELECT * FROM OLD2;
QUIT;


The INTERSECT command returns only those records that occur in both data sets.

                
PROC SQL;
    SELECT * FROM OLD1
    intersect
    SELECT * FROM OLD2;
QUIT;


Compare Union with Outer Union:

Union: produces all unique rows from both queries.

Outer Union: concatenates the series results.

                
PROC SQL;
    SELECT * FROM OLD1
    outer union
    SELECT * FROM OLD2
    ;
QUIT;


Note: SQL allows DUPLICATE variables names in VIEW but not in CREATE TABLE.

                
PROC SQL;
    CREATE TABLE TMP AS
    SELECT * FROM OLD1
    outer union
    SELECT * FROM OLD2
    ;
QUIT;

CREATE TABLE cannot handle duplicate variables: i.e.
WARNING: VARIABLE SUBJ already exists on file WORK.TMP.

Using SQL Table Aliases:

SQL aliases are used to give a database table or a column in a table, a temporary name -- to make column names more readable. SQL aliases can be useful when handling 2 or more tables. Table Alias allows one to create distinctions amongst the variables from different tables, so to better distinguish said variables without ambiguity.

                
PROC SQL;
    SELECT a.subj,a.age,b.subj,b.age,b.married as married
    FROM OLD1 a, OLD2 b
    WHERE a.subj=b.subj
    ;
QUIT;


In an INNER JOIN, only keeps observations with both key values that match in the selected variable, or variables.
     Using the table names as the alias.

                
PROC SQL;
    SELECT *
    FROM OLD1, OLD2
    WHERE old1.subj=old2.subj
    ;
QUIT;


Add the phrase "order by old1.subj" to put the table in Subject Order.

                
Proc SQL;
    SELECT *
    FROM OLD1, OLD2
    WHERE old1.subj=old2.subj
    ORDER by old1.subj
    ;
QUIT;
